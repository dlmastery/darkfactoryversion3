# Gate Policy

Default gates:

- Intake gate: goal, scope, owners, requirements, constraints, assumptions, and acceptance criteria are usable.
- Elaboration gate: architecture, NFRs, major risks, alternatives, and model decisions are reviewed.
- Construction gate: code, tests, implementation notes, and artifact updates are linked.
- Verification gate: tests, security, performance, reliability, traceability, and review evidence pass.
- Transition gate: deployment, rollback, observability, incident path, release notes, and maintenance guide exist.

Default thresholds:

- Governance compliance: 98 percent.
- Each selected expert reviewer: 96 percent.
- Mandatory traceability: 100 percent or explicit deferral.
- Critical/high security findings: zero unresolved.

Lowering a threshold requires a decision record and human owner.
