# Release Readiness

Required:

- Release scope.
- Deployment steps.
- Post-deploy verification.
- Rollback steps.
- Config and secrets handling.
- Observability: logs, metrics, traces, dashboards, alerts.
- Known risks and mitigations.
- Incident response path.
- Human owner.
- Support owner.
- Maintenance notes.
- Training or walkthrough needs.

Fail readiness if ownership, rollback, or observability is missing.
