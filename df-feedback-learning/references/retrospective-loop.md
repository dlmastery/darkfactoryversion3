# Retrospective Loop

For each review, failure, incident, or lesson:

1. Classify feedback.
2. Link it to affected requirements, artifacts, code, tests, gates, and owners.
3. Decide whether it creates a requirement, scenario, template update, rubric update, or governance update.
4. Implement the update.
5. Add regression evidence when the lesson came from a defect.
6. Record the decision and owner.
