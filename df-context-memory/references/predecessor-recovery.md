# Predecessor Recovery

Use this when work resumes after a pause, context compaction, role swap, failed gate, or human edit.

## Recovery Steps

1. Identify current Attractor Run Record, control graph node, and work-ledger item.
2. Load only the current node context pack and project book index.
3. Retrieve predecessor decisions, rejected alternatives, unresolved questions, and residual risks.
4. Compare predecessor state with current files or artifacts to detect drift.
5. Reconstruct the next safe action and owner.
6. Run a fresh-session replay: a new human or agent should be able to explain what to do next and what evidence is missing.
7. Record recovery evidence and revalidation triggers.

## Required Evidence

- Previous owner.
- Current owner.
- Source artifacts inspected.
- Decisions recovered.
- Drift found.
- Pending verification.
- Next action.
- Confidence and reason.

Escalate when predecessor state conflicts with current artifacts, production state, customer decisions, or standards.
