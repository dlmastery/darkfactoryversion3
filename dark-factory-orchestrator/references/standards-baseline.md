# Standards Baseline

Use this baseline unless the project provides a stricter company playbook.

- ISO/IEC/IEEE 12207:2017 for software life cycle processes. Current as checked on 2026-04-24.
- ISO/IEC/IEEE 15289:2019 for life-cycle information item content.
- ISO/IEC 42001:2023 for AI management system governance.
- NIST SP 800-218 SSDF 1.1 final for secure software development. Track SP 800-218 Rev. 1 / SSDF 1.2 draft as an update candidate.
- OWASP SAMM for software assurance maturity across governance, design, implementation, verification, and operations.
- OMG MDA for CIM/PIM/PSM modeling when model-driven architecture is required.
- OMG UML 2.5.1 when UML notation is used.
- CMMI V3.0 practices when maturity, measurement, supplier governance, or process improvement are requested.

Correction rule: if a user or transcript says ISO 12207:2026, treat it as an unverified shorthand. Use ISO/IEC/IEEE 12207:2017 until a newer official standard is confirmed.
