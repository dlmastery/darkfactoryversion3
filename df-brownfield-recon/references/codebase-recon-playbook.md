# Codebase Recon Playbook

Before changing an existing system:

1. Map repository structure and entry points.
2. Identify build, test, lint, typecheck, and run commands.
3. Read existing docs, configs, deployment files, and tests.
4. Locate ownership boundaries and architectural seams.
5. Identify user changes already present in the worktree.
6. Characterize current behavior before proposing changes.
7. Find impacted files, tests, artifacts, and operational procedures.
8. Produce a change impact record.

Never treat silence in documentation as permission to guess.
