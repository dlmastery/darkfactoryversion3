# Change Impact Template

Use this structure:

- Change request ID.
- Summary.
- Current behavior.
- Requested behavior.
- Impacted requirements.
- Impacted code areas.
- Impacted tests.
- Impacted artifacts.
- Impacted deployment or operations.
- Compatibility risks.
- Migration risks.
- Regression plan.
- Rollback plan if production-sensitive.
- Open questions.
- Recommended implementation slice.
